Bài test 6. Tạo ra 1 array các chữ số từ số startNum chuyền vào đến số endNum chuyền vào.

Sửa hàm buildArray(startNum, endNum) sao cho kết quả trả về là một array chứa tất cả các số từ số startNum đến số endNum.

  • Hàm này phải là hàm đệ quy, nghĩa là bên trong hàm này phải gọi lại chính hàm này.
  • Không được sử dụng các cú pháp loop như for, while hay các hàm nâng cao của loop như forEach, map, filter, reduce.
Ví dụ: buildArray(1, 5) phải trả về [1, 2, 3, 4, 5], buildArray(4,4) phải trả về [4]

Bài test 7. Kiểm tra 2 object bất kỳ trong 1 mảng có mối liên hệ với nhau.

Sửa hàm checkPath(arr, a, b) trong đó arr chuyền vào là 1 array có dạng như sau:
[
{id: 1, connectedTo: [7, 3]},
{id: 2, connectedTo: [8, 9]},
{id: 3, connectedTo: [1, 4]},
{id: 4, connectedTo: [7, 3]},
{id: 5, connectedTo: [7]},
{id: 6, connectedTo: [2]},
{id: 7, connectedTo: [1, 4, 5]},
{id: 8, connectedTo: [2, 9]},
{id: 9, connectedTo: [2, 8]}
]
a và b là 2 id của 2 object bất kỳ trong array trên.
Kết quả trả về là true nếu từ 2 objects này, ta có thể tìm được các mối liên hệ trực tiếp hoặc gián tiếp thông qua các arr connectedTo của từng object.
Mối quan hệ trực tiếp: Nếu object A có connectedTo chứa id của object B, 2 objects này được coi là có mối quan hệ trực tiếp
Mối quan hệ gián tiếp: Nếu object A có mối quan hệ trực tiếp đến một hoặc nhiều object khác và các object khác này lại có mối quan hệ trực tiếp đến một hoặc nhiều object khác và cứ tiếp tục như vậy đến khi có một object có mối liên hệ trực tiếp tới object B.
Ví dụ 1: checkPath(arr, 1, 5) sẽ trả về true vì từ object 1, ta thấy có mối liên hệ trực tiếp với object 7 và object 3, trong đó object 7 có mối liên hệ trực tiếp với object 5, vậy object 1 và object 5 có mối liên hệ, kết quả là true
Ví dụ 2: checkPath(arr, 2, 8) sẽ trả về true vì từ object 2, ta thấy có mối liên hệ trực tiếp với object 8
Ví dụ 3: checkPath(arr, 5, 2) sẽ trả về false vì từ object 5, ta không thấy có mối liên hệ trực tiếp cũng như gián tiếp đến object 2.
